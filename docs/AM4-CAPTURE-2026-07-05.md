## Capture-Confirmed Wire Shapes

Non-`PARAM_RW` startup/status functions:

- `fn 0x64` multipurpose ACK: observed as `f0 00 01 74 15 64 <fn> <rc> <cs> f7`; `rc=0x00` for OK.
- `fn 0x08` firmware/version request and response: response payload includes version bytes and build date.
- `fn 0x47` system-info-style request and response: same function number as gen-2 `GET_SYSINFO`; AM4
  payload is still opaque.
- `fn 0x12` mode switch requests are ACKed by `fn 0x64`.

`PARAM_RW` action families observed:

- `0x000d` long read: 64-byte response, `hdr4=0x0028`; this is the existing live bypass read shape.
- `0x000e` short read: 23-byte response, `hdr4=0x0004`; existing generic read shape.
- `0x0010` AM4-Edit live/value poll: 18-byte request, 23-byte response, `hdr4=0x0004`.
- `0x0012` preset-name read: 55-byte response, `hdr4=0x0020`; existing preset-name codec.
- `0x0017` descriptor read: 23-byte request, 64-byte response, `hdr4=0x0028`; payload still opaque.
- `0x001f` GET_PATCH: 238-byte descriptor response; existing device-true dirty-bit source.
- `0x0026` AM4-Edit status/zero poll: 18-byte request, 23-byte zero-valued response, `hdr4=0x0004`.
- `0x0027` and `0x0028`: rare AM4-Edit action variants; semantics still unassigned.

The AM4 diagnostic decoder now labels the newly confirmed shapes in
`src/devices/am4/describe.ts`, with capture-backed tests in `test/am4/describe.test.ts`.

## Telemetry Verdict

No separate gen-3-style AM4 telemetry channel appeared. Meter/tuner/CPU-like activity is carried as
ordinary AM4 `PARAM_RW` polling, especially action `0x0010`, plus status-ish action `0x0026`.

Current implementation stance:

- Safe to keep AM4 out of the gen-3 telemetry builders.
- Safe to say the fresh capture did not reveal push telemetry.
- Not yet safe to render AM4 meters/tuner/CPU from `0x0010` because the per-address semantics and
  value scaling still need correlation against screen/audio notes.
- Safe to expose capture-correlated AM4 live-poll address labels for diagnostics. These are labels, not
  a completed telemetry parser.
